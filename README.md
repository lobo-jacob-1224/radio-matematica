**Radio Matemática** es un proyecto de investigación avanzada de **Jacobo Cánovas García** que unifica la música clásica, la teoría de números y el procesamiento digital de señales (DSP) bajo el paradigma de la **Fisioforma**.

A diferencia de un reproductor de audio convencional, este sistema actúa como un **Moldeador Matemático** que aplica leyes físicas al flujo de audio en tiempo real para alcanzar un estado de **Gracia Matemática**.

## 🌀 Arquitectura SSS (S1, S2, S3)

El motor de procesamiento implementa tres fases de transformación física:

1.  **Fase S1: Resonancia de Bessel (Identidad)**: Estabiliza la entropía del flujo sonoro mediante funciones de Bessel de primera especie ($J_1$).
2.  **Fase S2: Horizonte de Riemann (Rectificación)**: Alinea las frecuencias armónicas con los **Ceros de Riemann** ($\rho$) en la Línea Crítica, transmutando el ruido en geometría sagrada.
3.  **Fase S3: Aniquilación de Fase (Saturación)**: Aplica una no linealidad hiperbólica ($tanh$) para añadir calidez analógica y normalización RMS, sincronizando el audio con el latido humano (Bio-Pulse 72 BPM).

## 🚀 Características Principales

-   **Rigor DSP v44.0**: Motor de convergencia de alta resolución (4096 puntos).
-   **Estéreo Binaural S3**: Desplazamiento de fase basado en la **Resonancia Schumann (7.83 Hz)**.
-   **Interfaz Cinematográfica**: Visualización en tiempo real del Orbe de Fourier y el Corazón Biológico.
-   **Persistencia SQL**: Registro automático de la telemetría cuántica en `riemann_zenit.db`.
-   **Sintonizador Global**: Acceso a las mejores estaciones de música clásica del mundo.

## 🛠️ Instalación y Uso

### Dependencias
- Python 3.13+
- NumPy, SciPy, Matplotlib
- FFMPEG (para la captura del stream)
- PipeWire/PulseAudio (`pw-play` o `aplay`)

### Ejecución
```bash
python3 engine.py
```

### Paquete Debian
También puedes instalar la versión profesional para Linux:
```bash
sudo dpkg -i radio-matematica_1.0.deb
```
